An Interface With Problems: Thermostats

Personas and Storyboarding

This project ultimately provides a strong framework of a "problem" that can be used for further design work to come up with a solution for creating simpler and more user-friendly thermostats.

A thermostat is a device used to detect temperature changes for the purpose of maintaining the temperature of an enclosed area essentially constant. It’s requirements to function are simple: a user needs to be able to set a desired temperature for the AC. Here is a sketch of a thermostat:

User Research

I started by recording observations while interviewing multiple thermostat users. I asked the following questions:

  • What temperature do you prefer to maintain at home?
  • Would you describe this interface as “simple”?
  • Do you know how to create a temperature schedule?
  • Do you know the difference between “cool” and “heat” when you set the thermostat to the same temperature?
  • Do you know the difference between the “system” and “fan” switches?
  • Do you know which rooms you are setting the temperature for?
  • How would you describe the thermostat?

Key Observations

  • The thermostat has multiple buttons that are never used.
  • Users are unsure of which rooms are affected by the thermostat.
  • Users are unsure of what each button does.
  • Users failed at knowing how to use each button, thus the interface failed the users.

Interview Themes

  • Users only want to set a temperature or turn off the system.
  • Context for which rooms are affected was missing.
  • There was inadequate visual feedback.
  • The interface is dated when compared to any modern technologies users interact with on a daily basis.
